이 문서는 사람이 읽는 설명용 문서다. 에이전트가 매번 참고할 최소 원문은 [`/codex/common-work-guidelines.md`]({{ "/codex/common-work-guidelines.md" | relative_url }})에 둔다.
{: .article-lead }

## 문서 역할

- `codex/common-work-guidelines.md`: 에이전트가 반복 참조하는 최소 지침. 짧게 유지한다.
- 이 문서: 왜 지침을 나눴는지와 운영 기준을 사람이 읽기 좋게 설명한다.
- 각 프로젝트의 `Design.html`: 프로젝트별 목표, 기획 의도, UX/아트 방향, 결정 사유를 둔다.

## 운영 기준

- 반복 실행에 필요한 규칙은 먼저 `codex/common-work-guidelines.md`에 짧게 쓴다.
- 설명, 배경, 예외 사례처럼 길어지는 내용은 이 문서나 별도 문서로 분리한다.
- HTML을 직접 길게 쓰기보다 Markdown 원본을 우선한다.
- 에이전트용 지침과 사람용 설명이 다르면 에이전트용 raw Markdown이 실행 기준이다.

## 현재 방향

공통 지침은 "작업 전에 반드시 떠올릴 것"만 남긴다. 세부 컨벤션, 프로젝트별 판단, 아트 방향, 기획 문서 작성 기준은 각각의 전용 문서로 보낸다.
